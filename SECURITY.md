# Security Policy

## Reporting a vulnerability

If you find a security issue, please do not open a public issue with exploit details.

Instead, report it privately to the repository maintainer or organization owner.

Include:

- A short description of the issue
- Steps to reproduce it
- Affected files, components, or workflows
- Any suggested mitigation, if known

## Scope

This organization is primarily used for educational, research, and prototype work. Security-sensitive information such as secrets, tokens, credentials, private keys, and personal data should never be committed.
